概要

Open Knowledge Format（OKF） は、Andrej Karpathy の LLM-wiki パターンを、ポータブルで相互運用可能なフォーマットに定式化したオープン仕様。Google Cloud の Data Cloud チームが 2026 年 6 月に発表した。OKF v0.1 は、知識を「YAML frontmatter 付き Markdown ファイルのディレクトリ」として表現し、異なる producer が書いた wiki を異なるエージェントが翻訳なしで消費できるよう、少数の合意された規約だけを定める。

詳細

解決する課題

組織内の知識（テーブルのスキーマ、メトリクスの意味、runbook、API の deprecation 通知など）は、メタデータカタログ・wiki・コードコメント・一部シニアエンジニアの頭の中、といった断片化した互換性のないシステムに散在している。各エージェント開発者が同じ context 収集問題を毎回ゼロから解いており、知識は生成元のサーフェスに閉じ込められている。OKF は「新しいサービス」ではなく「フォーマット」でこれを解決する。

設計

  • bundle = concept（テーブル、データセット、メトリクス、playbook、runbook、API など）を表す Markdown ファイルのディレクトリ。1 concept = 1 ファイルで、ファイルパスが concept の identity。
  • 各ファイルは構造化フィールド用の小さな YAML frontmatter（type / title / description / resource / tags / timestamp）と Markdown 本文を持つ。
  • concept 同士は通常の Markdown リンクで結び、ディレクトリを関係の graph にする。任意で index.md（progressive disclosure）と log.md（変更の時系列）を含められる。
  • 「Just markdown / Just files / Just YAML frontmatter」が特徴で、複雑な圧縮方式・新しい runtime・必須 SDK は不要。

3 つの設計原則

  1. Minimally opinionated: 全 concept に要求するのは type フィールドのみ。相互運用の表面だけを定義し、content model は producer に委ねる。
  2. Producer/consumer independence: 誰が書くかと誰が消費するかを分離。人間が書いた bundle を AI が消費、といった組み合わせが自由。
  3. Format, not platform: 特定のクラウド・DB・モデルプロバイダ・エージェントフレームワークに依存しない。読み書きに専有アカウントや SDK を要求しない。

同梱物

参照実装として、BigQuery データセットを走査して concept ドキュメントを起草・enrich する enrichment agent、任意の bundle を自己完結 HTML の interactive graph にする static HTML visualizer、3 つの sample bundle（GA4 e-commerce / Stack Overflow / Bitcoin）を公開。spec・repo・sample は GitHub の GoogleCloudPlatform/knowledge-catalog で入手でき、Google Cloud の Knowledge Catalog も OKF の ingest に対応した。

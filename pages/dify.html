Dify

LLMアプリ開発・運用のオープンソースプラットフォーム「Dify」を、基本要素から実装まで解説した記事（yushibats, 2025年11月）の要約。

概要

Dify は「LLM を使ったアプリケーションの開発と運用を手軽に行えるオープンソースプラットフォーム」。BaaS（Backend as a Service）と LLMOps の考え方を組み合わせ、認証・DB・ストレージ・開発UI などAIアプリに必要なバックエンド一式を提供する。単なる RAG チャットボット構築ツールではなく、AIエージェントを作り・動かし・運用できる Agentic AI 開発プラットフォームとして設計されている。

詳細

提供元と背景

  • 提供元: LangGenius Inc.（創業者 Luyu Zhang、元 Tencent Cloud DevOps）
  • 2023年5月に OSS 公開。GitHub スターは公開1年で10万超、LangChain を一時追い抜く勢い。
  • 名称は「Define + Modify」＋「Do it for you」。2025年5月に読みを「ディファイ」から「ディフィ」へ変更。
  • 活用事例: サイバーエージェント（社員20%利用、月3,000時間超削減）、IIJ、カカクコム/食べログ、東京都、町田市など。

AIエージェントの整理

記事は前提として AIエージェントを段階分けする: ワークフロー型（決められた手順を直線実行）→ シングルエージェント型（AIが判断しながらタスク遂行）→ マルチエージェント型（複数AIが役割分担・協調）。「Agentic AI」はマルチエージェント型に近い、複数エージェントが協調して目的達成する体系を指す。2025年時点では多くの企業がワークフロー型にとどまり、エージェント導入はPoC段階。

Dify の特徴

  • OSS による柔軟性・信頼性: ソース確認・カスタマイズ可能、閉域環境デプロイ可、Community Edition は無償・商用利用可。
  • 複数LLM対応とコスト最適化: 40超のプロバイダー対応。用途に応じたモデル切替、トークン使用量・APIコストの可視化。
  • ノーコード/ローコード構築: プロンプト・ツール呼び出し・分岐処理をビジュアルエディタで組み立て。
  • 展開性・標準化: フローやエージェントを DSL 形式でエクスポートし他部署へ展開。

提供形態と基本構成

  • クラウド版（Sandbox 無料 / Professional $59月 / Team $159月）、セルフホスト版（Community / Premium / Enterprise）。
  • 基本画面は スタジオ（アプリ作成）/ ナレッジ（RAG向けナレッジベース）/ ツール・プラグイン。
  • アプリの型: チャットボット / チャットフロー / ワークフロー / エージェント / テキストジェネレーター。
  • LLM追加や機能拡張は「プラグイン」形式でマーケットプレースからワンクリック追加。

実装例: 「資料見つかるくん」

PowerPoint資料の横断検索アプリ。ナレッジベースが .pptx 非対応・図内テキスト検索が困難という制約に対し、ファイル投入アプリと検索アプリの2段構成を採用。PPT→PDF→PNG 変換後、画像を Embedding モデル（Cohere Embed v4）でベクトル化して Oracle Autonomous DB に保存し、画像ベースで検索。OCI 上（Compute + Autonomous Database + OCI Generative AI + Object Storage）に構築。

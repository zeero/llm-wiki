tmux-portal

Methods

tmux-portal

複数の AI コーディングエージェント（Claude、Aider、Cursor 等）のセッションを効率管理する、TPM 互換の tmux プラグイン。その実装指示書。

概要

tmux-portal は tmux plugin manager (TPM) 互換のプラグインで、AI エージェントセッションの高速切り替え、存在しないセッションの自動作成、新規ウィンドウでのコマンド実行、セッション単位の status line カスタマイズを提供する。技術スタックは Bash、テストは Bats（Bash Automated Testing System）。この文書は AI コーディングエージェントに実装させることを想定した作業指示書として書かれている（末尾に「ユーザーが作成した未検証コンテンツ」の注記あり）。

詳細

ディレクトリ構成

tmux-portal/
├── portal.tmux            # TPM エントリポイント（実行可能）
├── scripts/tmux-portal.sh # メインスクリプト
├── tests/                 # test_helper.bash, mocks/tmux_mock.bash, tmux-portal.bats
├── README.md
└── LICENSE

主要仕様

  • portal.tmux: TPM が *.tmux を自動実行する。#!/usr/bin/env bash で開始し、${BASH_SOURCE[0]} からプラグインディレクトリを解決。キーバインドは設定せずユーザーに委ねる。
  • tmux-portal.sh: -s/--session, -c/--command, --status-style, -h/--help を解析。主要関数は session_exists（has-session）、create_session（new-session -d）、switch_to_session（$TMUX の有無で switch-client か attach-session）、show_session_switcher（fzf があれば fzf、なければ select）、create_window_with_command、set_status_style。
  • メインロジック: オプションなし→スイッチャーで選択して切替、コマンドのみ→スイッチャーで対象選択、セッション未存在→自動作成、コマンド指定→新規ウィンドウで実行。

テスト（Bats）

tmux コマンドを全てモック化し、実際の tmux セッションを操作しない。モックは連想配列 MOCK_SESSIONS 等で状態を保持し、MOCK_TMUX_CALLS に呼び出しを記録してアサートする。テスト間で状態をリセット。git submodule で bats-core / bats-support / bats-assert を導入。

実装フェーズと注意

Phase1 基本構造 → Phase2 コア機能 → Phase3 拡張機能 → Phase4 テスト → Phase5 ドキュメント の順を推奨。TPM 互換のため *.tmux は実行可能属性が必要。hooks 相当の重い処理や stdout 汚染を避ける設計は、AI エージェントセッション管理の文脈で重要。

Figma MCP と SwiftUI コード生成

Figma のデザイン情報を Framelink Figma MCP で取得し、Cursor と Claude で SwiftUI コードへ変換する際の試行錯誤。

概要

記事では Cursor 1.1.4 と Claude 4 を使い、GitHub アプリのプロフィール画面を題材に、Figma の URL を渡して SwiftUI を生成している。単純に URL を渡すだけでもコードは得られるが、デザインとの差分が残るため、MCP が実際に返す情報を調べてプロンプトと Figma 側の構造を調整している。

MCP が返す情報の限界

Framelink Figma MCP には、ファイル全体のレイアウト情報を取得する get_figma_data などのツールがある。一方で、記事の検証では、Component 内の Icon の選択状態や Style/Variable の意味など、Figma 上で見えている情報が取得データに十分現れない例が示されている。取得できる構造と取得できない意味を分けて確認しないと、生成コードの見た目や状態がずれる。

実装時の要点

  • Figma の URL を渡すだけで完成とせず、対象ノードとデザインデータの内容を確認する。
  • Component、Icon、Typography、Color、Style/Variable の表現が MCP の出力に含まれているかを確認する。
  • 生成された SwiftUI のレイアウト、色、フォント、状態、画像を元デザインと比較する。
  • 欠落した情報は、Figma の構造を整理するか、プロンプトで明示して補う。

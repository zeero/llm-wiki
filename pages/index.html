技術系

Concepts 11 pages

  • Multi-Agent Orchestration
    複数の専門特化した AI エージェントを協調動作させ、分業と統合で1つの成果物を作るアーキテクチャ。2026 年の AI 開発は単体エージェントからこのマルチエージェント・オーケストレーションへシフトした。
  • Loop Engineering
    コーディングエージェントを手でプロンプトする代わりに、「エージェントをプロンプトするシステム（loop）」を設計する実践。レバレッジの中心が「プロンプトを書くこと」から「プロンプトを出す仕組みを設計すること」へ移動したという主張。
  • RAG（Retrieval-Augmented Generation）
    RAG は、LLM 単体では知り得ない社内情報や最新情報を、外部から検索して回答生成時に注入する仕組み。「世間知らずな AI 本部長を、8 人の専門スタッフで固めた過保護なオフィス」という比喩で全体像を掴める。
  • LLM Memory Architecture
    LLM チャットアシスタントがユーザーの情報を「記憶」しているように見せるためのコンテキスト設計。ChatGPT の実測例では、vector database や会話履歴への RAG を使わず、性質の異なる4層を毎回のプロンプトに合成することで実現している。
  • Swift Concurrency: MainActor 上かの判定
    Swift 6 の非同期コードで「メインスレッドか」ではなく「どの actor 上か」を問い、`MainActor.assertIsolated()` などで MainActor 実行を検証する方法。
  • LLM-wiki
    LLM エージェントが読み書きする知識ベースを、YAML frontmatter 付き Markdown ファイル群として構造化するパターン。Andrej Karpathy の原案に由来する。
  • Transformer
    自己注意（self-attention）機構を中核とするニューラルネットワークアーキテクチャ。2017 年の論文「Attention Is All You Need」で提案され、現代の大規模言語モデルの基盤となっている。
  • Prompt Caching
    LLM API で、繰り返し使う大きなコンテキストをキャッシュし、再入力時のトークン費用とレイテンシを削減する仕組み。
  • Prompt Engineering の代表的手法
    LLM に与える指示の構成を工夫し、出力の精度・一貫性・再現性を高めるための手法群。
  • UITableView の基本
    UIKit の `UITableViewController`・`UITableViewCell`・delegate/data source の役割と、セル表示・選択処理の基本をまとめる。
  • UIViewController のライフサイクル
    View の読み込み・表示・非表示・破棄に対応する `UIViewController` の主要メソッドを整理する。

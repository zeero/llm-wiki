JitRL 式メモリ（Claude Code）

JitRL の考え方を応用し、Claude Code に過去の成功/失敗パターンを記憶させて「学習するAIエージェント」にする仕組み。

概要

Claude Code は標準では CLAUDE.md によるメモリを持つが、これは手動更新で、成功/失敗の文脈や類似状況の検索を持たない。JitRL 式メモリは、各インタラクションを Policy Triplet <State, Action, Outcome> として自動記録し、ベクトル類似度で過去の経験を検索して現在のプロンプトへ注入する。勾配更新（fine-tuning）を行わず、経験リプレイとアドバンテージ重み付けで成功パターンを優先する点が特徴で、arXiv の JitRL（Just-in-Time Reinforcement Learning for LLM Agent）論文に基づく。

配布形態は2種類が紹介されている。ひとつは babushkai/jitrl-skill として公開された Claude Code Plugin（2コマンドでインストール、Hooks 自動登録）、もうひとつは ~/.claude-jitrl/ 以下に自前で構築する完全手順である。

詳細

Policy Triplet とスコアリング

  • State: 現在のコンテキスト（読んだファイル、エラー内容、目標）
  • Action: Claude の行動（Edit / Write / Bash 等の tool 呼び出し）
  • Outcome: 結果（成功/失敗、ユーザーフィードバック）

スコアは成果から算出する。成功=+5、「perfect/great」フィードバックで+5、部分的成功=+2、失敗だが学びあり=-2、完全失敗=-5、といった加減点で数値化する。

アドバンテージ計算

action タイプごとの平均スコアから全体平均（baseline）を引いた値を「アドバンテージ」とし、プラスのアクションを推奨、マイナスのアクションを回避対象として提示する。

Advantage(Edit) = 平均スコア(Edit) − 全体平均スコア

Hooks 統合

3つの Claude Code ライフサイクル Hook を利用する。

Hook タイミング 役割
UserPromptSubmit プロンプト送信時 類似経験を検索し「成功パターン / 失敗パターン / 推奨アプローチ」をコンテキスト注入
Stop 応答完了時 インタラクション詳細を記録
SessionEnd セッション終了時 評価してストレージに保存

ストレージと検索（自前実装）

  • ディレクトリ: ~/.claude-jitrl/{experiences,indexes,cache}。経験は episodes.jsonl、ベクトルは Faiss（IndexFlatIP）で保持。プロジェクトパスの MD5 ハッシュで名前空間を分ける。
  • Embedding: OpenAI text-embedding-3-small（1536次元）。API 不在時は語のハッシュによるフォールバック embedding。結果は .npy でキャッシュ。
  • 検索: 正規化した query ベクトルで内積検索し、類似度 threshold（例 0.6〜0.7）以上を採用。類似度とスコアの加重（例 0.3:0.7）で並べ替える。
  • 依存: faiss-cpu, numpy, openai, click。設定は config.yaml（gamma, similarity_threshold, TTL 等）。
  • コマンド: /jitrl:init, /jitrl:stats, /jitrl:search [query]（プラグイン版）。

既存手段との比較

CLAUDE.md / claude-mem / claude-supermemory と比べ、JitRL 式は「成功/失敗スコアリング」「アドバンテージ計算」「推奨アプローチ提示」を持つ点が差別化とされる。導入効果として「同じ説明の繰り返し約60%減」等の実測が挙げられているが、著者による自己申告値である。

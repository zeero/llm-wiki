AI コーディングのワークフロー

AI コーディングエージェントを既存プロダクトや Issue 起点の開発へ組み込むための、計画・実装・検証・レビューの流れ。

設計ファイルとタスクリスト

Cline と GitHub Copilot Agent を既存プロダクトで使う例では、タスクリストとタイムスタンプ付きの設計ファイルを clinerules_bank/tasks/ に残す。計画した内容、編集開始時刻、実施結果を分離して記録することで、長い作業の進捗と判断の履歴を追いやすくする。

Issue から Pull Request まで

GitHub Actions と Codex CLI を組み合わせる例では、codex ラベル付き Issue を起点に次の流れを自動化する。

  1. リポジトリを checkout し、Git と Node.js の実行環境を準備する。
  2. Codex CLI をインストールし、Issue のタイトルと本文からプロンプトを作る。
  3. Issue 番号に基づくブランチで codex -a full-auto --quiet を実行し、修正とコミットを行う。
  4. 変更を push し、gh pr create で Pull Request を作成する。
  5. 必要に応じてビルドやテストを実行し、生成された変更を人間が確認する。

安全境界

自動化では、Issue 本文をそのまま実行権限へ渡さないこと、API キーを Secrets で管理すること、Actions の permissions を必要最小限にすることが重要である。full-auto は便利だが、変更対象・Git 操作・テスト・PR の範囲をワークフローとプロンプトで明示し、レビューを省略しない。

SWE-RL

オープンソースソフトウェアの進化データとルールベース報酬を用いて、実世界のソフトウェアエンジニアリング向けに LLM の推論を強化学習でスケールさせる手法。Meta（facebookresearch）による NeurIPS’25 採択研究。

概要

SWE-RL は、実世界のソフトウェアエンジニアリングに対して強化学習ベースの LLM 推論を初めてスケールさせたアプローチ。open-source software evolution データと rule-based reward を活用する。論文「SWE-RL: Advancing LLM Reasoning via Reinforcement Learning on Open Software Evolution」（arXiv:2502.18449、NeurIPS’25）の公式コードベースが facebookresearch/swe-rl として公開されている（Python 100%、主に UniverseFly=Yuxiang Wei が実装）。

詳細

報酬関数

現状のコードは prompt テンプレート（src/swerl/core/prompts.py）と、系列類似度に基づく報酬関数の実装（src/swerl/core/reward.py）を提供する。3 つの reward API があり、いずれも戻り値は [0, 1] の範囲:

  1. calculate_search_replace_reward: search/replace 変更と oracle 変更の類似度（論文で使用）
  2. calculate_reward_unidiff: 2 組の unified diff 間の類似度
  3. calculate_reward: 任意の編集フォーマットに使える汎用版

LLM 出力は <think> と <solution> で囲み、solution 内に search/replace ブロックを書く形式。

Agentless Mini

Agentless を土台に、openai-python での高速非同期推論、スケーラビリティ向上のリファクタ、file-level localization のみ実施（修復には全ファイル内容を使用）、複数の reproduction test による reranking 対応、といった改良を加えたパイプライン。任意の OpenAI 互換エンドポイントで動作し、vLLM や SGLang で自前ホストできる。localization → repair → rerank の全パイプラインを提供し、SWE-bench Verified / Lite での評価に対応（sharding で並列化可能）。reproduction test 生成・回帰テスト選択・テスト実行はリファクタとインフラ差異のため WIP。

LLM-wiki

LLM エージェントが読み書きする知識ベースを、YAML frontmatter 付き Markdown ファイル群として構造化するパターン。Andrej Karpathy の原案に由来する。

概要

LLM-wiki は、組織や個人の知識を、人間にも LLM にも扱いやすい「YAML frontmatter 付き Markdown ファイルのディレクトリ」として管理する設計パターン。1 つの概念を 1 ファイルで表し、ファイル同士を相互リンクして知識のグラフを作る。索引としての index.md と、変更の時系列を残す log.md を伴うことが多い。この Vault の Wiki/ 自体がこのパターンの実装であり、schema も「Karpathy 原案準拠」と明記している。

詳細

基本構造

  • 1 concept = 1 Markdown ファイル: ファイルパスが概念の identity になる。
  • frontmatter: type / title / description / tags / timestamp などの構造化メタデータ。
  • 相互リンク: 概念同士を Markdown リンク（WikiLink）で結び、ディレクトリを関係グラフにする。
  • index.md / log.md: 段階的開示（progressive disclosure）のための索引と、追記型の変更ログ。

発展形

Google Cloud はこの LLM-wiki パターンを、producer/consumer 非依存で相互運用可能な仕様に定式化した Open Knowledge Format (OKF)（OKF）を発表した。OKF は「サービスではなくフォーマット」として、異なるエージェントが翻訳なしで同じ知識を消費できることを狙う。

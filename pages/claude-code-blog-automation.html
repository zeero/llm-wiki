Claude Code でのブログ記事執筆自動化

Claude Code のスラッシュコマンドとエージェント機能で、インタビュー → 執筆 → レビューのブログ記事執筆ワークフローを仕組み化した事例。

概要

tacoms の SRE はぶちん氏が、記事の構成を考えてから書き始めるまでのハードルを下げるため、Claude Code で記事執筆を仕組み化した事例。@shinpr_p の OSS ai-coding-project-boilerplate にインスパイアされ、ブログ執筆に特化したエージェントとワークフローを実装した。記事自体もこの仕組みで書かれている。

詳細

プロジェクト構造

.claude/commands/（newpost.md, review.md）、.claude/agents/（interviewer.md, ghostwriter.md, editor.md）、docs/writing-style.md（スタイルガイド）、templates/、drafts/、articles/ で構成。

エージェントの役割

/newpost tech <テーマ> で執筆ワークフローが始まる。

  1. インタビュアー（自動実行）: 記事に必要な情報を段階的にヒアリング。質問に答える形で思考が整理され、方向性が定まる。
  2. ゴーストライター（自動実行）: 集めた情報を基に、docs/writing-style.md のスタイルガイドに従って執筆。
  3. 編集者（/review で必要時に呼び出し）: 文体チェックとファクトチェック。当初は 3 エージェントを順に自動実行する想定だったが、初稿後に通常のやり取りが挟まる実態に合わせ、編集者は独立コマンド /review として切り出した。エージェント連携は Claude Code の Task ツールで実現。

スタイルガイドの重要性

自分らしい文章にはスタイルガイドが要。冒頭の挨拶パターン、文体ルール（です・ます調、「〜んです」より「〜ました」、メタ的内容は「！」で強調）、締めの定型文などを writing-style.md に定義しておくと、挨拶から締めまで自動で入る。AI が書いた文章で自分の文体に合わない点（語尾、くどい自己紹介、季節の話題など）は気づきとして次回ルールに反映し、使いながら精度を上げていく。

結果と工夫

LT スライドの下書きを渡すとイメージ通りの記事が約 1 時間で完成。音声入力ツール VoiceInk（買い切り・ローカル動作・Mac のみ）と組み合わせてインタビュー回答を入力。はてなブログ API による自動投稿は会社ブログへの個人アカウント投稿が難しくコピペ対応となった。

OpenClaw

メッセージングアプリからローカルの AI コーディングエージェントを操作できる、オープンソースの自律型 AI パーソナルアシスタント。

概要

OpenClaw（旧称 Clawdbot → Moltbot）は、WhatsApp・Telegram・Slack・Discord・iMessage・LINE などのメッセージングプラットフォームと、ローカルで動く AI コーディングエージェントを、中央の Gateway 経由でつなぐパーソナル AI アシスタント。自分のデバイス上で動作し、会話とコードをプライベートに保つ。Peter Steinberger 氏によって 2025 年末に開発され、2026 年初頭に GitHub 10 万スター超を獲得した MIT ライセンスの OSS。「自律型」で、話しかけられて返すだけでなく AI が自分で考えて動き、永続メモリや自己改善（自分でスキルを書く）を持つ点が特徴。

詳細

アーキテクチャ

  • Gateway: 中核となる長時間実行プロセス（デーモン常駐）。メッセージングとの接続を管理し、主ポート 18789 で WebSocket（ws://127.0.0.1:18789）と HTTP ダッシュボード（http://127.0.0.1:18789/、Docker セットアップの接続先）を、ポート 18793 で HTTP/Canvas（/__openclaw__/canvas/）を提供。設定は ~/.openclaw/openclaw.json。デフォルトで localhost のみ listen。
  • Pi Agent: 実際の推論とタスク実行を担当。Gateway から RPC 経由で呼ばれ、選択した AI モデル（Claude、GPT、ローカルモデル等）と通信。ユーザーごとに独立したセッションを保持（DM は共有メイン、グループは隔離）。
  • Skills / Tools: ファイルシステム・シェル実行・ブラウザ制御・外部 API 連携などの機能を提供する拡張モジュール。TypeScript/JavaScript で書かれ @openclaw/* スコープの npm パッケージとして配布。100 以上のコミュニティ製スキルがある。

インストールと実行

ワンライナー（curl -fsSL https://openclaw.ai/install.sh | bash）または npm（npm install -g openclaw@latest）で導入し、openclaw onboard --install-daemon でセットアップウィザードを起動。Node.js v22 以上が必要。Ollama と組み合わせる場合は ollama launch openclaw で起動でき、少なくとも 64k トークンの context length が推奨される。qwen3-coder、glm-4.7、gpt-oss、cloud モデルの kimi-k2.5・minimax-m2.1 などが OpenClaw と相性が良い。

セキュリティ

OpenClaw はファイルシステムフルアクセスとシェル実行権限を持つ「高権限インフラ」として扱う必要がある。主なリスクはプロンプトインジェクション、アカウント乗っ取り、悪意あるスキル、設定ミス。推奨対策:

  • DM ポリシーを pairing（未知の送信者にペアリングコード）または allowlist にする（open は本番非推奨）
  • グループは requireMention: true でメンション時のみ反応
  • Gateway の Control UI 認証を有効化、トークンはヘッダー認証（Authorization: Bearer）
  • API キーは環境変数や暗号化ファイルで管理し Git にコミットしない
  • 最小権限の原則、サンドボックス/専用アカウントで運用、--host 0.0.0.0 での公開露出を避け、リモートは SSH トンネル/VPN 経由
  • openclaw doctor で露出や設定を定期診断

ターミナル操作・サーバー運用・セキュリティ理解のある人向けで、「ワンクリック AI 秘書」を求める層には不向きとされる。

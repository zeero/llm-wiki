Slack ブックマーク自動付与（n8n + Socket Mode）

特定の Slack チャンネルへの投稿を検知し、n8n 経由で bookmarks.add を呼んで自動でブックマークを付与するシステムの設計書。

概要

Slack チャンネルへのメッセージ投稿イベントを Socket Mode（WebSocket）で受信し、n8n のワークフローから Slack API bookmarks.add を呼び出してブックマークを追加する。公開 URL やトンネリング（ngrok 等）が不要で、ローカル環境でも開発できるのが Socket Mode 採用の理由。無料版 Slack ではワークフロービルダーが使えないため、既存アプリを流用する前提。

詳細

方式比較

  • ポーリング: 定期 API 呼び出しでリソース無駄・レート制限リスク・リアルタイム性低。
  • Event Subscriptions（Webhook）: リアルタイムだが n8n の公開 URL とトンネリングが必要。
  • Socket Mode（採用）: 公開 URL 不要、リアルタイム、ローカル開発可。

Slack App 設定

  • Socket Mode を有効化し App-Level Token（xapp-...、scope connections:write）を作成。
  • Event Subscriptions で message.channels（またはプライベートは message.groups）を購読。
  • Bot Token Scopes: channels:history, channels:read, bookmarks:write。
  • 取得する認証情報: Bot User OAuth Token（xoxb-...）、App-Level Token（xapp-...）、Signing Secret。

n8n ワークフロー

構成は [Socket Mode Trigger] → [HTTP Request (bookmarks.add)]。コミュニティノード（例: @mbakgun/n8n-nodes-slack-socket-mode）を導入。HTTP Request は POST https://slack.com/api/bookmarks.add、Body に channel_id / title / type: "link" / link を渡す。title は投稿者名やメッセージ先頭でカスタマイズ可能。

運用上の注意

  • トラブルシュート: Socket Mode 接続エラー（App-Level Token・有効化確認）、ブックマーク失敗（bookmarks:write scope・Bot のチャンネル招待確認）、イベント未受信（購読設定・Bot メンバー確認）。
  • セキュリティ: Token/Secret は環境変数か n8n の認証情報管理で保管、定期ローテーション、最小権限。

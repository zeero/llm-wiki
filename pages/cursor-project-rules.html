Cursor の Project Rules

Cursor のルールファイルを分割・参照し、編集中のファイルやドメインに応じて適切な指示を適用する方法。

@file によるルール分岐

Project Rules では、最初に読むメインルールから別のルールファイルを @file 記法で参照できる。メインルールを入口にして、編集中のファイル内容やプロジェクトの状況に応じたドメイン別ルールへ案内すると、条件分岐のような構成を作れる。

main.mdc
├── user-domain.mdc
└── payment-domain.mdc

ルールを一枚に詰め込まず、共通の入口と専門領域の指示を分離することで、AI に渡すガイダンスを状況に合わせて管理しやすくなる。ルールの適用結果は実際の編集内容で確認し、参照関係が循環しないようにする。

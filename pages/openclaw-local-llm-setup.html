OpenClaw × ローカル LLM 構築ガイド

OpenClaw を完全ローカル LLM 環境（Ollama / llama-server）で運用するための構築手順。Docker / 非 Docker、Linux / macOS / Windows(WSL2) 対応。

概要

API 利用はすぐに上限に達する可能性があるため、OpenClaw をローカル LLM で運用する構成をまとめた手順書。バックエンドは Ollama（簡単・高速）と llama-server（llama.cpp / OpenAI 互換・本格運用）の両対応。推奨アーキテクチャは Browser/CLI → OpenClaw Gateway → Ollama または llama-server。

詳細

OpenClaw の構築

  • Docker（推奨）: git clone https://github.com/openclaw/openclaw.git → cd openclaw → ./docker-setup.sh（build・onboard・Gateway 起動・token 発行を自動化）。ダッシュボードは http://127.0.0.1:18789/、token 再表示は docker compose run --rm openclaw-cli dashboard --no-open。
  • ローカルインストール（非 Docker）: macOS/Linux は curl -fsSL https://openclaw.ai/install.sh | bash、Windows(PowerShell) は iwr -useb https://openclaw.ai/install.ps1 | iex（Windows は WSL2 + Ubuntu 推奨）。初期設定 openclaw onboard --install-daemon。

ローカル LLM① Ollama（簡単・安定）

curl -fsSL https://ollama.com/install.sh | bash でインストール、ollama pull llama3.3 / qwen2.5-coder:32b 等でモデル取得。export OLLAMA_API_KEY="ollama-local" で接続し、設定で model: { primary: "ollama/llama3.3" } を指定。確認は ollama list / openclaw models list。

ローカル LLM② llama-server（llama.cpp・本命）

llama.cpp 公式の OpenAI 互換 API サーバ（/v1/chat/completions・/v1/models・streaming 対応）で、本格 GPU 運用・複数起動向け。CPU 起動例は ./llama-server -m models/*.gguf --port 8080 --ctx-size 8192 --threads 8、GPU は --n-gpu-layers 999 --ctx-size 16384。起動確認 curl http://127.0.0.1:8080/v1/models。OpenClaw 側は OPENAI_API_KEY（ダミー可）と OPENAI_BASE_URL="http://127.0.0.1:8080/v1" を設定し model: { primary: "openai/..." } を指定。systemd で常駐化（Restart=always、LimitNOFILE=1048576）が超重要とされる。

運用・トラブル・セキュリティ

  • トラブル: モデルが出ない→llama-server 未起動 / /v1 抜け / ポート不一致。応答が遅い→ctx-size 過剰 / GPU layers 不足 / swap 不足。
  • セキュリティ: skill/plugin は必ずコード確認、Docker/VM 分離推奨、API キーは .env 管理、外部公開時は Cloudflare Access 等。
  • おすすめ運用: 日常利用→Ollama、重作業/コーディング→llama-server、複数モデル切替→複数 llama-server + ポート分離、公開→Cloudflare Tunnel + Access。

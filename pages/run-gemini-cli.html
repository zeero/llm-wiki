run-gemini-cli

Gemini CLI を GitHub ワークフローに統合する公式 GitHub Action。PR レビュー・Issue トリアージ・コード分析を自律／オンデマンドで実行する。

概要

run-gemini-cli（google-github-actions/run-gemini-cli）は、Gemini を Gemini CLI 経由で開発ワークフローに統合する GitHub Action。定型的なコーディングタスクを担う自律エージェントとしても、@gemini-cli fix this issue のように会話的に呼び出せるオンデマンドの協働者としても機能する。GitHub リポジトリ内で PR レビュー、Issue トリアージ、コード分析・修正などを実行できる。本ソースは v0.1.21 の README。

詳細

特徴

  • Automation: Issue オープンなどのイベントや nightly などのスケジュールでワークフローを起動
  • On-demand Collaboration: Issue / PR コメントで @gemini-cli /review のようにメンションして起動
  • Extensible with Tools: Gemini の tool-calling で GitHub CLI（gh）など他 CLI と連携
  • Customizable: リポジトリ直下の GEMINI.md でプロジェクト固有の指示・コンテキストを与える

クイックスタート

  1. Google AI Studio で Gemini API キーを取得
  2. リポジトリの Secret に GEMINI_API_KEY として登録
  3. .gitignore に .gemini/ と gha-creds-*.json を追加
  4. ワークフロー設定: Gemini CLI で /setup-github（推奨）、または examples/workflows の定義を手動コピー（中央ディスパッチャの gemini-dispatch.yml も必要）

ワークフローと認証

提供される主なワークフロー: Gemini Dispatch（イベント/コマンドに応じて振り分ける中央ディスパッチャ）、Issue Triage、Pull Request Review、Gemini CLI Assistant（汎用会話アシスタント）。認証は Google 側が Gemini API キーまたは Workload Identity Federation、GitHub 側がデフォルト GITHUB_TOKEN またはカスタム GitHub App（推奨）。Vertex AI / Gemini Code Assist も選択でき、Google Cloud への OpenTelemetry テレメトリ送信による observability にも対応する。

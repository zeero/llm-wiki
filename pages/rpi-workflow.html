RPI（Research → Plan → Implement）ワークフロー

Claude Code で大きな機能開発を進めるための 3 段階ワークフロー。調査・計画・実装を分離し、早期の GO/NO-GO 判定で無駄な実装を防ぐ。

概要

RPI ワークフロー は、Research（調査）→ Plan（計画）→ Implement（実装）の 3 フェーズで機能開発を進める手法。shanraisshan/claude-code-best-practice リポジトリで紹介されており、claude-code-best-practice リポジトリ の中核パターンの一つ。大きな機能・チーム開発向けで、個人・小規模タスクには重すぎるとされる。

詳細

各フェーズ

  • /rpi:research: 実現可能性を分析し、GO / NO-GO を判定する。NO-GO なら早期に止め、無駄な実装を避ける。
  • /rpi:plan: pm.md・ux.md・eng.md・PLAN.md などの計画ドキュメントを自動生成する。
  • /rpi:implement: フェーズごとに実装し、code-reviewer がレビューする。

位置づけ

「Research で NO-GO なら止める」設計により、着手前に見込みの薄い開発を弾ける点が特徴。ベストプラクティス集では、チーム開発・大規模機能で有用とされ、個人開発では過剰になりうると整理されている。

execute-n8n-workflow スキル開発仕様

Docker コンテナの OpenClaw から、ホスト側で動く n8n のワークフローを Webhook トリガーで実行するエージェントスキルの開発仕様書。

概要

ユーザーが自然言語で「〜のワークフロー動かして」と指示したとき、適切な n8n ワークフローを特定して Webhook をトリガーする Node.js 製スキル execute-n8n-workflow の仕様。外部依存なし（標準ライブラリのみ、fetch は Node 18+、fs でファイル操作）でシンプルさを優先する。

詳細

アーキテクチャ

  • 環境変数: N8N_BASE_URL（例 http://host.docker.internal:5678）、N8N_API_KEY（認証）、N8N_WORKFLOW_REPO_PATH（ワークフロー JSON のリポジトリパス）。
  • ワークフロー JSON 構造: 1 ファイル 1 ワークフロー。ルートは配列 [{ nodes: [...], ... }] で要素は常に 1 つ。Webhook トリガーのパスは [0].nodes[].parameters.path。

処理フロー

  1. 環境変数の検証（欠けていればエラー出力して終了）。
  2. N8N_WORKFLOW_REPO_PATH 配下の .json を再帰取得し、ファイル名や JSON 内のワークフロー名で候補を絞る。
  3. LLM が指示から最適候補を推測し 必ず確認を挟む（候補 1 つ→実行可否、複数→リスト提示、なし→見つからない旨）。
  4. 選択された JSON の [0].nodes[] から node.type === 'n8n-nodes-base.webhook' かつ node.parameters.path を持つノードを探しパスを特定。
  5. Webhook を POST で実行。
  6. 結果返却（2xx 成功→「実行しました」、その他→ステータスコードとエラーメッセージ）。

スキル構成

skills/execute-n8n-workflow/ 配下に SKILL.md（metadata に openclaw の requires.env として 3 環境変数、primaryEnv を指定）と trigger.js。trigger.js はコマンドライン引数で指示を受け取り、環境変数から設定を読み、JSON を探索・パースし、結果を標準出力（OpenClaw が読み取る）へ、エラーは標準エラー出力へ、終了コードは成功=0/失敗=1。

エラーハンドリングと注意

環境変数不足→設定案内、リポジトリパス不正→確認促し、JSON パースエラー→ファイル名と詳細を表示しスキップして継続、Webhook パス未発見→構成確認、リクエスト失敗→ステータスコードとレスポンスボディ表示。非機能要件はシンプルさ優先・トリガー成功のみ確認（ポーリングなし）・fetch デフォルトタイムアウト依存・最小限のログ。OpenClaw は Docker 内で動くためホストアクセスに host.docker.internal を使い、確認プロンプトは安全性優先で必須。将来拡張としてパラメータ渡し・実行結果ポーリング・実行履歴管理・連鎖実行が挙げられている。

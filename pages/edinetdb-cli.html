edinetdb CLI

有価証券報告書から抽出した日本の全上場企業 約4,000社の財務データを、コマンド1本で引ける Python パッケージ。EDINET DB サービスに CLI という入口を追加したもの。

概要

pip install edinetdb でインストールできる Python パッケージ／CLI。有報から抽出した日本の全上場企業（約4,000社）の財務データをコマンド1本で引ける。

pip install edinetdb
edinetdb auth login
edinetdb financials E02144 --years 5

EDINET DB（https://edinetdb.jp）は既に API Key 約7,000件・月間約100万リクエストで運用中のDBサービスで、そこに CLI を追加した形。既存の API Key はそのまま流用できる。ソースは https://github.com/Cabocia/edinetdb-cli 。

位置づけ（3本柱の1つ）

EDINET DB は有報データの構造化ベースに AI 分析・スクリーニング・大株主分析・不動産クロスDB連携を乗せたデータインフラ。従来 3 経路（Web UI、REST API、MCP Server）があり、CLI はその中間を埋める。

  • Jupyter/Python スクリプトから → subprocess.check_output(["edinetdb", "financials", "..."]) で雑に呼べる
  • cron や GitHub Actions から → edinetdb rankings roe --format csv > daily.csv 1行でOK
  • curl のヘッダ設定が面倒 → edinetdb search toyota で即結果
  • MCP 非対応の AI エージェント → Bash ツールで edinetdb を叩くだけで全機能利用可能

MCP Server（AI エージェント向け）／REST API（アプリ向け）／CLI（ターミナル向け）の3本柱を揃えるのが狙い。

主なコマンド

全27エンドポイント分のコマンドが揃う（edinetdb --help）。

  • edinetdb search toyota — 企業検索（Rich テーブル表示）
  • edinetdb financials E02144 --years 5 — 財務時系列（revenue, operating_income, net_income, total_assets, eps …）
  • edinetdb rankings roe --limit 10 — ROE ランキング
  • edinetdb screener -w 'roe:gte=15' -w 'revenue_cagr_3y:gte=10' — スクリーナー
  • edinetdb shareholders search "三菱UFJ信託" — 大株主逆引き
  • edinetdb text-blocks E02144 --full — 有報のテキストブロック（事業リスク等の全文）
  • edinetdb status — データ鮮度確認（status, search は認証不要で即試せる）

設計判断

  • Typer + httpx + rich の3点セット: 依存を最小化。Flask や SDK 自動生成は使わず薄く手書き。CLI の UX は Typer 標準で十分、HTTP は httpx で同期/非同期両対応、出力整形は rich が鉄板。
  • 認証: J-Quants CLI の OAuth デバイスフローと迷ったが、既に API Key が多数稼働している実情から既存の API Key paste 方式を採用。環境変数（EDINETDB_API_KEY）と named profile（複数 Key 使い分け）に対応。credentials は ~/.config/edinetdb/credentials に chmod 0600 で保管。
  • 出力形式: JSON / table（デフォルト、rich table）/ CSV を切替可能。
  • エラー → exit code マッピング: シェルスクリプトでの扱いやすさ優先。0=成功、1=未分類、2=認証エラー(401/403)、3=レート制限(429)、4=Not Found(404)、5=パラメータ不正(400)、6=サーバーエラー(5xx)。

OpenAPI 3.0 spec 同時公開

CLI 公開に合わせ API ドキュメントを OpenAPI 3.0 準拠にリニューアル。openapi.yaml を Single Source of Truth とし、Flask 実装との突合を scripts/verify_openapi.py で CI チェック（ドリフトがあればデプロイを止める）。Redoc UI は openapi.json から自動描画、CLI は x-codeSamples 経由で右パネルにコマンド例を表示、他言語 SDK（TypeScript / Go / Rust）の生成も可能。

J-Quants CLI との比較

CLI 化の参考にした JPX の J-Quants CLI から、コマンド階層・複数出力形式を取り入れた。意図的に変えた点: 分野範囲（J-Quants は市場データ中心、edinetdb は有報の構造化で広い）、AI 活用前提（MCP Server を既に提供し CLI はシェル/パイプ派の補完）、認証 UX（edinetdb は既存 API Key 生態系を優先）。

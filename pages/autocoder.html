AutoCoder

Claude Agent SDK ベースの、長時間・複数セッションに対応した自律コーディングエージェント。仕様を渡すとアプリを 24 時間作り続ける実行基盤。

概要

AutoCoder（leonvanzyl/autocoder）は、Claude Agent SDK 上に構築された自律コーディング基盤。AI コーディングツールが抱える「セッション分断」問題（コンテキスト限界による品質低下、中断時のゼロからの説明し直し、長時間実行の監視負担、状態の消失）を、状態の永続化によって解決する。仕様（Spec）を渡して放置すれば、中断と再開を繰り返しながら数時間〜数日かけてアプリを完成まで作り上げることを狙う。

詳細

Two-Agent Pattern

  • Initializer Agent: プロジェクト開始時に 1 回だけ実行。Spec を読み込み Feature（機能単位）に分解し、各 Feature の詳細なテストケース（通常 200 件以上）を生成、features.db に保存し、プロジェクト構造と Git を初期化。所要 10〜20 分以上。
  • Coding Agent: 初期化後に繰り返し実行。未完了 Feature を DB から 1 件取得 → 実装 → テスト → 検証 → passing マーク → 必要に応じ回帰テスト → アトミックな Git コミット。1 Feature あたり 5〜15 分。

永続化と Feature 管理

すべての状態（Feature 一覧、状態、テストケース、依存関係）を SQLite の features.db に保存するため、セッションが切れても失われず、任意のタイミングで中断・再開できる。エージェントは MCP ツール（feature_get_stats / feature_get_next / feature_mark_passing / feature_skip / feature_get_for_regression / feature_create_bulk）経由で進捗を制御し、暴走を防ぐ。

利用と安全設計

Claude Code CLI と Claude 認証（Pro/Max または API キー）が前提。UI モード（./start_ui.sh、ブラウザで Kanban ボード・WebSocket ライブ出力）と CLI モード（サーバー/SSH 向け）がある。多層防御として Bash コマンドの許可リスト、プロジェクト外・センシティブパスへのアクセスブロック、OS レベルのサンドボックス（コンテナ/VM 推奨）、UI でのリアルタイム監視・即時停止を備える。生成物は通常のアプリとして修正・デプロイ・継続開発が可能。

他ツールとの比較

  • AutoCoder: 長時間自律実行・仕様からの完全自動構築が得意（Two-Agent + DB 永続化、人間の関与は仕様入力のみ）
  • GSD: コンテキスト劣化の防止、大規模プロジェクト管理（タスク分散 + サブエージェント）
  • Superpowers: TDD・レビュー・品質重視のワークフロー徹底

3 者は異なるレイヤーで機能するため併用可能（Superpowers で設計 → GSD でフェーズ分割 → AutoCoder で自動実装）。限界として Claude 環境前提、生成品質が Spec とプロンプト設計に強く依存、完全自動でも監視と微調整は必要。

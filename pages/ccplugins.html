CCPlugins

Claude Code CLI 向けのコマンド集「CCPlugins」（brennercruvinel / notlikeDev）のGitHub READMEの要約。

概要

CCPlugins は Claude Code CLI を拡張する24個のプロフェッショナルコマンド集。「毎回『シニアエンジニアのように振る舞って、過剰設計しないで』とタイプするのに疲れた開発者」が作ったもので、繰り返しの開発タスクで週2〜5時間の節約をうたう。MIT ライセンス、GitHub スター2.7k。Opus 4 / Sonnet 4 向けに最適化。

詳細

コマンド群（24個）

  • 開発ワークフロー: /cleanproject（デバッグ成果物をgit安全付きで削除）, /commit（Conventional Commits）, /format, /scaffold, /test, /implement, /refactor
  • 品質・セキュリティ: /review（マルチエージェント分析）, /security-scan, /predict-issues, /remove-comments, /fix-imports, /find-todos, /create-todos, /fix-todos
  • 高度な分析: /understand, /explain-like-senior, /contributing, /make-it-pretty
  • セッション・プロジェクト管理: /session-start, /session-end, /docs, /todos-to-issues, /undo

設計思想

  • 会話的コマンド: 一人称（”I’ll help…“）でClaudeの協調的推論を活性化。
  • ビルド非依存の指示: ハードコードされたツールを使わず、どこでも動く。
  • ネイティブツールのみ: Grep / Glob / Read / Write / TodoWrite / Task（サブエージェント）を活用。
  • 安全第一: 破壊的操作の前に自動 git チェックポイント。
  • セッション継続性: /implement や /refactor はプロジェクトルートに専用フォルダ（plan.md / state.json）を作りセッションを跨いで状態を保持。
  • コマンドは ~/.claude/commands/ に markdown で配置し、$ARGUMENTS で引数を受け取る。git 連携コマンドは AI attribution（Co-authored-by 等）を付与しない設計。

V2 開発中

Anthropic の最近のアップデート（SDK改善、ネイティブサブエージェント、コンテキストメモリ、CLI強化）により一部の slash command が冗長化。V2 は「Claude Code がまだ効率的にできないこと」に絞ったアーキテクチャ再設計となる予定。現行 v1 は安定・機能的なまま。

Claude Code への「省察」実装（メタ認知）

メタ認知（思考についての思考）のフレームを使い、Claude Code に判断を言語化・振り返りさせる「省察の層」を実装するアプローチ。

概要

人間が学習できるのは、タスクをこなす自分とは別に、それを観察するもう一人の自分（メタ認知）がいるからである。ジョン・フレイヴェルはメタ認知を「思考についての思考」と定義し、「モニタリング」と「制御」に分けた（Flavell, 1979）。またドナルド・ショーンの省察的実践家（Schön, 1983）は、省察を「状況との対話」と捉える。これらを AI エージェントに実装し、タスク実行後に結果を振り返って自分のやり方を問い直させる。

詳細

省察の層 reflection/

記憶が「水平の層（何を覚えるか）」、スキーマが「世界の層（どう動くか）」だとすれば、メタ認知は「垂直の層」。タスクを実行する自分と観察する自分を構造として分ける。

~/.claude/
├── CLAUDE.md
├── memory/       # 記憶の層（What happened）
├── schemas/      # 世界の層（How things work）
└── reflection/   # 省察の層（How I think）
    ├── REFLECTION.md    # インデックス
    ├── decisions-log.md # 判断と理由のログ
    └── patterns.md      # 抽出された思考パターン

2つの仕組み

  1. 判断の言語化（行為の中の省察）: 重要な判断のたびに「選択・理由・代替案・確信度（5段階）」を decisions-log.md に追記。確信度を数値で残すのがポイントで、低い数値は後で見返すべき判断の目印になる。
  2. パターンの抽出（行為についての省察）: 週次で decisions-log.md を読み返し、確信度が低かった判断やうまくいかなかったケースの共通点を patterns.md へ「昇格」させる。記述は「パターン・根拠・適用・例外」の4点で書き、適用と例外をセットで持たせることで表層的な模倣を防ぐ。

モニタリングと制御の運用

  • モニタリング = 判断の確信度の自己申告（自分の認知状態を自分で観察）。
  • 制御 = 新しい判断の前に patterns.md を参照し、過去の失敗を繰り返していないか確認。

これらを CLAUDE.md に「判断後は decisions-log に記録」「判断前に patterns.md を参照」「週次で昇格」と書く。毎回読まれるインデックス（REFLECTION.md）は薄く保ち、詳細は各ファイルに逃がす（目次は薄く、中身は厚く）。昇格基準は「同じパターンが3回以上再現」「適用と例外が言語化できる」。

意味づけ

繰り返し現れる判断パターンが「思考の型」として昇格し、個別の判断がやがて方法論になる。この「昇格」構造は記憶・スキーマ・省察で一貫する。考え方を AI に渡そうとした瞬間に、自分がそれを言語化できていなかったことに気づく——考え方を渡すことは、考え方を知ることでもある。

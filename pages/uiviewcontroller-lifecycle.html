Concepts

UIViewController のライフサイクル

View の読み込み・表示・非表示・破棄に対応する UIViewController の主要メソッドを整理する。

読み込み

loadView はコードから View を構築する段階、viewDidLoad は View が読み込まれた直後に呼ばれる。初期設定や一度だけ行う View の構成は、この読み込み段階に置く。

表示と非表示

  • viewWillAppear: 画面に表示される直前
  • viewDidAppear: 画面への表示が完了した直後
  • viewWillDisappear: 画面から外れる直前
  • viewDidDisappear: 画面から外れた直後

viewDidDisappear が呼ばれても View Controller オブジェクトが破棄されたとは限らない。delegate の破棄や通知の解除をこのタイミングだけで判断すると、再表示時の挙動を壊す可能性がある。

破棄と古い API

最終的なオブジェクト破棄に関係する処理は dealloc の責務であり、画面が一度消えたこととは分けて考える。viewDidUnload は古い iOS の View 解放時 API で、現在の実装の前提にしない。

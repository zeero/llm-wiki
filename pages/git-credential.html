Git HTTPS 認証情報の保存

HTTPS 経由の Git 操作で毎回パスワードを入力しないための credential helper の選択肢を整理する。

選択肢

  • git-credential-cache: 常駐プロセスに一時的に記憶させる
  • git-credential-osxkeychain: macOS のパスワード管理を使う
  • git-credential-store: ファイルに保存するが、パスワードが平文になる
  • Git-Credential-Manager-for-Windows / git-credential-gnomekeyring: 各 OS のパスワード管理を使う

記事では cache を使う設定として次を示している。

git config --global credential.helper cache

.netrc や store のように認証情報を平文で保存する方式は避け、利用環境の安全な credential manager を優先する。これは古い Git 記事の整理であり、現在のホスティングサービスが要求する token や認証方式は別途確認する。

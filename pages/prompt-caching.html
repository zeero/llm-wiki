Prompt Caching

Concepts

Prompt Caching

LLM API で、繰り返し使う大きなコンテキストをキャッシュし、再入力時のトークン費用とレイテンシを削減する仕組み。

概要

Prompt Caching（プロンプトキャッシュ） は、システムプロンプトや長大な背景情報など、リクエスト間で共通する入力コンテキストをサーバー側でキャッシュし、後続リクエストで再利用することで課金トークンと処理時間を減らす機能。Anthropic API をはじめ主要 LLM プロバイダが標準提供している。

詳細

有効な場面と限界

プロンプトキャッシュは、短時間（数分〜数十分程度）のうちに同じコンテキストへ連続アクセスする用途で効果を発揮する。一方、次のような場合はキャッシュが揮発・不一致になり効きにくい。

  • コーディングで数時間空けて断続的に質問する場合（キャッシュ有効期限を超える）。
  • RAG のように検索結果テキストが毎回入れ替わり、コンテキストが一致しない場合。

代替・補完手段との関係

pxpipe（プロンプト画像化によるトークン圧縮）（pxpipe）は、こうしたキャッシュが効かない静的で巨大な背景知識を PNG 画像化して渡すことで、有効期限に依存しない固定の低コストを狙う手法。ただし精度低下の代償があるため、まずプロンプトキャッシュのような基本策を導入した上で、読み落としを許容できるワークロードにのみ切るべきカードとされる。

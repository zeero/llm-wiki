Claude Code 強化ツール集

Claude Code を中心とした開発を高速化・安定化するための周辺 CLI・プラグイン・拡張ツールを、用途別に整理したまとめ。

概要

Claude Code などの AI エージェントはコード生成そのものは高速だが、Git 操作・リポジトリ移動・パッケージ管理・レビュー返信といった周辺作業や、フォーマット揺れに惑わされる編集・危険なシェルコマンドの生成などがボトルネックになる。ここでは 3 つの記事で紹介された「Claude Code を底上げするツール」を横断的にまとめる。ツールは大きく (1) エージェント自身がパス上から呼んで使う CLI、(2) 人間のレビュー・運用を助けるツール、(3) Claude Code の設定・セッション・GUI・並列化を支えるエコシステム、に分類できる。

詳細

「構造で扱う」CLI スタック（エージェントが自分で呼ぶ）

Bozhidar Batsov の記事「Supercharging Claude Code with the Right (CLI) Tools」で紹介された 10 本を、teppyama が Claude Code 視点で解説したもの。鍵は「コードを単語の並び（正規表現）ではなく構文の構造として扱う」発想。ツールを CLAUDE.md に明記して指示するのではなく、パス上に置いておくだけで Claude が自分で気づいて使い始める運用が推奨されている（CLAUDE.md の容量とコンテキストを節約できる）。

  • 高優先（構造で扱う中核）
    • ast-grep: 構文パターンでコードを検索・書き換え。$PROP && $PROP() → $PROP?.() のように、同じメタ変数を 2 回書けば「同じ識別子の繰り返し」を構造的に縛れる。コメント・文字列リテラルの誤マッチを避けられる（tree-sitter ベース、Rust 製）。
    • difftastic: 行指向ではなく構文構造で diff を取る。インデント・改行だけの整形変更を色で抑え、本筋の変更だけをレビューに残す。AI 生成差分のレビューを読みやすくする（30 以上の言語対応、Rust 製）。
    • shellcheck: AI が大量生成するシェルの危険（未定義変数を踏む rm -rf $UNSET/ 等）を実行前に静的検出。AI と実行の間に「動かす前のフィルタ」を 1 段はさむ（Haskell 製）。
  • 中優先
    • sd: モダンな置換ツール。sed の正規表現エスケープ地獄を回避。
    • scc: コード行数・構成を高速把握。プロジェクトの規模感を Claude に伝える。
    • yq: YAML/JSON/TOML/XML の AST ベースクエリ。コメント・順序を保持したまま設定を書き換えられ、再生成による事故を防ぐ。
    • comby: 言語非依存の構造化置換（ast-grep より単純なパターン向き）。
  • 補助
    • hyperfine: コマンド実行のベンチマーク。
    • watchexec: ファイル変更で自動コマンド実行（テスト駆動を組ませやすい）。
    • delta（git-delta）: git diff のシンタックスハイライト。

一括導入コマンド: brew install ast-grep difftastic shellcheck sd scc yq comby hyperfine watchexec git-delta。まず全部入れて 1〜2 週間眺め、Claude が選んで使うものを見てから削るのが負荷の少ない手順とされる。

周辺ワークフローツール（人間の手数を減らす）

Findy Media の記事（Ubie 鹿野氏）で紹介された、開発フロー全体の速度を落とさないためのツール群。

  • Tool Search Tool: MCP ツールを遅延読み込み（loaded on demand）し、セッション開始時のコンテキスト消費をゼロにする。~/.claude/settings.json の env で ENABLE_TOOL_SEARCH: "true"。ツール選択精度も向上。
  • Claude-Mem: セッション間でコンテキストを自動保存・復元するプラグイン。作業を AI で圧縮し次セッションに関連コンテキストを注入。過去セッションは localhost:37777 で閲覧可能。
  • ccresume: 過去セッションをインタラクティブに閲覧・再開する CUI ツール（標準の claude --resume より使いやすい）。
  • PR コメント自動返信: gh api repos/{owner}/{repo}/pulls/{pr}/comments -X POST -f body="..." -F in_reply_to={comment_id} を覚えさせ、CodeRabbit/Copilot 等のレビューにスレッド形式で返信させる。
  • ni: ロックファイルを検出して npm/yarn/pnpm/bun/deno を自動判定（ni, ni pkg -D, nr dev）。
  • ghq + peco: ghq でリポジトリを ~/git/github.com/{owner}/{repo} 構造で管理し、peco で絞り込んで一発移動（g エイリアス）。
  • Raycast: ランチャー。ショートカットでアプリ起動やシェルスクリプト実行（複数エージェントの並列起動用スクリプト等）。

Claude Code エコシステム（設定・可視化・GUI・並列化）

Qiita のまとめ記事（flowernotfound）で紹介された、成熟した Claude Code コミュニティ発のツール群。

  • コスト・ログ可視化: ccusage（コスト・使用状況可視化、statusline 対応）、Claude Code Log（会話ログ閲覧、TUI/ブラウザ）、Sniffly（利用ログ分析ダッシュボード）。
  • 設定・ステータスライン: ccexp（設定ファイル管理）、ccstatusline / Claude Powerline（カスタムステータスライン）、Claude Code Templates / Cookbook / My Claude Code Project’s Starter Settings（設定テンプレート）、Claude Code Development Kit（開発環境セットアップ）。
  • 拡張フレームワーク・ワークフロー: SuperClaude Framework（全部入り拡張）、CCPlugins（定型タスクのカスタムコマンド集）、ccpm（GitHub 統合の仕様駆動開発）、Claude Code Spec Workflow（Kiro 統合の spec 開発）、Claude Code Workflows、Claude-Flow（オーケストレーション）、TDD Guard（TDD 強制）。
  • GUI / エディタ統合: opcode（旧 Claudia、デスクトップ GUI）、Claude Code Viewer / UI / Web UI / Chat（ブラウザ・VSCode・モバイル）、claudecode.nvim（Neovim）、Claude Code IDE for Emacs、cui（音声入力対応 GUI）、happy。
  • モデル・検索・並列: Claude Code Router（他モデル・ローカルモデル利用）、Claude Context（セマンティック検索 MCP、serena 的）、Claude Squad（エージェント並列化）、n8n-MCP（n8n 統合）。
  • サブエージェント・レビュー: Claude Code Subagents Collection / Awesome Claude Code Subagents（テンプレート集）、Claude Code Security Reviewer（セキュリティ脆弱性レビュー）、awesome-claude-code（情報集約リポジトリ）。
